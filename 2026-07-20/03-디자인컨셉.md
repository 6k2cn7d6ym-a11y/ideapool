## [A] 커뮤니티·동호회 통합 관리 앱

### 컨셉 방향

**톤**: 신뢰형 (규칙 존중) + 심리적 안도감 (투명성)  
**핵심 감정**: "투명하고 공정한 총무가 여기 있다" — 회비는 돈이라 신뢰부터.

### 타이포그래피

- **헤드라인** (금액·기한·인원): 보울드, 큼직함. 숫자가 가장 먼저 눈에 들어와야 함.
  - 예: "이달 납부율 85%" → "85%" 두드러짐
  
- **본문** (이름, 메모): 정상 웨이트, 읽기 편한 크기. 색상은 그레이.

- **라벨** (상태 표시): 작고 명확. "미납", "부분납", "완료" 등 상태를 한눈에.

### 디자인 원칙

1. **숫자가 주인공** — 텍스트 비중 최소, 숫자·상태 강조. 총무는 글을 읽으러 오는 게 아니라 "몇 명이 언제까지" 알러고 온다.

2. **색상으로 상태 전달** — "빨강=긴급", "초록=완료", "주황=주의" 이 셋만 쓴다. 일관성.

3. **한 화면에 "다음 액션 1개"만** — "지금 뭐할까" 고민 없게. 미납이 보이면 "알림 보내기" 버튼, 정산이 보이면 "결과 복사" 버튼.

4. **터치 영역이 명확** — 버튼, 카드, 행 모두 누르기 좋은 크기(최소 48px). 사무실에서 손가락으로 재빨리 조작.

## 달리 → A안 비평

**좋은 것:**
- 대시보드 숫자 위계 탁월. "85% / 17명 / 3명"이 3초 안에 들어온다.
- "즉시 조치 필요" 섹션 — 총무의 실제 워크플로를 정확히 짚음. 총무가 원하는 게 결국 "지금 뭐할까"이니.
- 정산 화면 디테일. "내야 할 금액 빨강 / 받을 금액 초록" 직관적.

**문제점:**
- 빨강이 너무 많다. 홈·회원 목록·정산 — 세 화면 전부에서 레드가 지배. 총무 혼자 보는 화면이어도 "빨간 앱" 이미지가 쌓이면 사용 피로 온다.
- "즉시 조치 필요"가 홈 화면을 장악하면, 납부 잘 하는 회원 17명의 기여가 사라진다. 앱이 "미납 관리 툴"로만 느껴지는 부작용.
- 참고 앱(토스·오늘의집·당근)이 섞여 있어서 결이 모호함. 세 앱은 무드가 꽤 다르다.

→ **동의.** 그리드는 15명 이하에 한정. 16명 이상은 자동으로 리스트뷰 전환하는 조건부 렌더링으로 처리.

**예상 지적 2:** "남았어요" 프레이밍이 총무에겐 너무 부드러울 수 있음. 긴장이 안 잡힌다.  
→ **부분 동의.** 홈은 달성 프레임 유지. 단, 미납자가 전월 이월되는 경우(2개월 연속 미납)는 별도 강조 처리. "3명 남음"과 "전월 이월 2명" 구분.

**예상 지적 3:** 최근 소식 피드가 쌓이면 노이즈.  
→ **동의.** 최신 3건만 노출, "더 보기" 뒤로 숨김.

## 합의안 — 최종 확정

**A안에서 가져오는 것:**
1. 대시보드의 숫자 위계 (85% / 17명 / 3명 구조)
2. 미납자 카드에 "알림 보내기" 버튼 직접 붙이기
3. 정산 화면의 내야 할/받을 금액 컬러 코딩

**B안에서 가져오는 것:**
1. 컬러 팔레트 — 진파랑 대신 웜 앰버+포레스트 (빨강은 미납 아이콘에만 국소적으로)
2. "X명 남았어요" 달성 프레임 (빨강 강조 전에 달성률이 먼저)
3. 정모 일정 카드를 홈에 올리기 (총무 단독 도구 → 모임 허브)
4. 미납 리스트를 하단 서랍형으로 — 홈이 "문제 목록"으로 시작하지 않게

**조정된 포인트:**
- 빨강(→뮤티드 로즈)은 아이콘·상태 뱃지에만. 섹션 전체를 물들이지 않는다.
- 그리드 뷰는 15명 이하. 이상이면 리스트뷰 자동 전환.
- 섹션 제목: "즉시 조치 필요" → "확인 필요 · N명에게 알림 보내기"

### 확정 핵심 원칙

1. **달성률이 먼저, 미납이 나중** — 홈은 "85% 완납"으로 시작. 미납은 그 아래.
2. **숫자가 주인공이되, 위협하지 않는다** — Bold 대신 SemiBold. 정보는 명확하게, 감정은 차갑지 않게.
3. **총무의 다음 액션은 한 번에** — 미납자 카드에 "알림 보내기" 하나만. 복잡한 선택지 없이.
4. **회원도 쓸 수 있는 화면** — RSVP, 납부 확인 — 총무 전용 아닌 모임 허브.
