## [7] AI 수업 리포트 + 학생 관리 앱

### 컨셉 방향

**톤**: 프로페셔널 + 효율성  
**핵심 감정**: "번거로운 일은 앱이 처리한다" — 선생은 수업에 집중, 리포트는 AI가 완성

개인 과외 선생(1~3명 고객)을 위한 앱. "빠르고, 정확하고, 번거롭지 않게"가 전부. 따뜻함보다는 **명확함**. 한번 써보고 바로 카카오톡 리포트를 붙여넣기까지 30초.

- **헤드라인** (학생명, 월): 세미볼드, 명확함. 보자마자 "이 학생"인지 알아야 함.
  
- **본문** (수업 메모, 생성된 리포트): 일반 웨이트, 읽기 좋은 크기. 복사하기 쉽게 라인 높이 넉넉하게.

- **라벨** (상태 표시): 작고 명확. "✅ 납부", "❌ 미납" 한눈에.

│  ─────────────────────────────────  │
│                                     │
│  ✅ 이 선생        100,000         │
│     마지막 수업: 7/21              │
│     [메모 입력]                    │  ← 이 버튼 클릭 → 화면 2로
│                                     │
│  ✅ 박 학생        120,000         │
│     마지막 수업: 7/19              │

**포인트**
- 화면 상단에 "이번 달 수강료 요약" — 선생이 가장 먼저 알고 싶은 정보.
- 학생 리스트는 상태(✅/❌)로 시작 — 미납자가 눈에 띄게.
- "메모 입력" 버튼이 각 학생마다 있음 — 학생 선택 후 추가 탭 없이 바로 입력.

- 최근 리포트 목록 아래로 스크롤 (추가 탭 없음).

### 디자인 원칙

1. **메모 → 리포트 → 복사가 전부** — 3단계가 한 화면에. 다른 건 없다.

2. **숫자와 상태를 먼저 보여주기** — 선생은 "지금 몇 명이 냈는가?"를 가장 먼저 본다.

3. **복사하기 만큼 중요한 버튼 없음** — 생성 다음은 복사. 복사 버튼은 항상 명확하고 큼.

4. **카톡으로 끝낸다** — "앱 → 복사 → 카톡 붙여넣기"가 핵심 워크플로우. 앱 내 발송·이메일·알림톡 제거.

5. **데스크톱 최적화** — 책상에서 "빠르게 입력 후 복사". 하지만 모바일도 한손 조작 가능하게 여백을 넉넉하게.
